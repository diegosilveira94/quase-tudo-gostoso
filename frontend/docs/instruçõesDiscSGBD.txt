Atividade em Grupo: Implementação do Cadastro no "Quase Tudo Gostoso"
Objetivo: Desenvolver uma aplicação funcional que permita cadastrar no banco de dados Quase Tudo Gostoso: usuários, categorias de receitas e refeição. A atividade visa exercitar habilidades de programação, modelagem de banco de dados e boas práticas de desenvolvimento em equipe. O formulário deverá capturar os dados, validá-los e inseri-los de forma segura em um banco de dados.



Usar o modelo de banco de dados já criado em sala de aula. As páginas não precisam estar relacionadas (podem estar desconexas).



Número de Participantes: 2 a 4 alunos.

Forma de entrega: .zipado com todo o código em um readme com a instrução de execução e o nome dos participantes.

A equipe deve escolher uma linguagem de programação para o backend (ex: Python com Flask/Django, JavaScript com Node.js/Express, PHP, etc.) e tecnologias para o frontend (HTML/CSS puro, ou com algum framework como Bootstrap).
Dicas

Desenvolver a tela de cadastro (cadastro.html) contendo o formulário com todos os campos definidos.

Utilizar os tipos de input corretos do HTML5 (ex: type="text", type="email", type="number", textarea).

Implementar validações básicas no lado do cliente (frontend) para garantir que campos obrigatórios sejam preenchidos (required).

Estilizar o formulário para que tenha uma aparência limpa e organizada.

Criar o "servidor" ou o script que receberá os dados do formulário (via método POST).

Estabelecer a conexão com o banco de dados criado.

Implementar uma função que receba os dados do formulário e execute uma instrução SQL INSERT para adicioná-los à(s) tabela(s) correspondentes.

Após a inserção, o sistema deve retornar uma mensagem de sucesso para o usuário (ex: "Dados cadastrado com sucesso!").

Garantir que o formulário do Frontend está enviando os dados corretamente para o endpoint do Backend.

Realizar testes completos:

Tentar submeter o formulário com campos em branco.

Tentar inserir dados com tipos incorretos (ex: texto no campo de número).

Cadastrar registros diferentes e verificar no banco de dados se todas foram inseridos corretamente.

Testar a inserção de textos com caracteres especiais e apóstrofos (ex: Bolo d'água).



Atenção: todos devem saber o código, pois apresentarão para a turma e individualmente para o professor, que poderá perguntar exclusivamente para algum aluno



Critérios de Avaliação
Funcionalidade (50%): A aplicação cadastra os dados no banco de dados sem erros? As validações funcionam?

Qualidade do Código (20%): O código está organizado, comentado e legível? A estrutura do projeto faz sentido?

Avaliação individual do Trabalho em Equipe (30%): Houve uma clara divisão de tarefas? A integração entre frontend e backend foi bem-sucedida?

Ao final, a equipe deve apresentar a aplicação funcionando e entregar o código-fonte para análise.
